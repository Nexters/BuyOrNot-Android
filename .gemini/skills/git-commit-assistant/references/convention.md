# Commit Convention

## Structure
`<type>/#<issue-number>: <subject>`

## Types
- `feat`: 새로운 기능 추가
- `fix`: 버그 수정
- `docs`: 문서 수정
- `style`: 코드 포맷팅, 세미콜론 누락 등 (코드 변경 없음)
- `refactor`: 코드 리팩토링
- `test`: 테스트 코드 추가/수정
- `chore`: 빌드 업무, 패키지 매니저 수정
- `add`: 에셋 추가 등
- `ci`: CI 관련 설정
- `build`: 빌드 관련 파일 수정
- `perf`: 성능 개선

## Subject Rules
- Max 50 characters.
- Start with an Uppercase letter.
- No period or special characters at the end.
- Use imperative mood, present tense (e.g., "Add", not "Added").

## Example
`feat/#10: 네트워크 에러 공통 Error View 컴포넌트 구현`
